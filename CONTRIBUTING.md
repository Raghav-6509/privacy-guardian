## ğŸ“ Contribution Guidelines

### Code Style
- **Python**: Follow PEP 8 guidelines
- **JavaScript**: Use consistent indentation (2 spaces)
- **React**: Follow React best practices and hooks conventions
- **CSS**: Use BEM methodology for class naming

### Commit Messages
Use clear, descriptive commit messages:
```
feat: add new privacy metric calculation
fix: resolve heat map rendering issue
docs: update API documentation
test: add unit tests for privacy score calculation
```

### Pull Request Process
1. Ensure your code follows the project's style guidelines
2. Test your changes thoroughly
3. Update documentation if necessary
4. Submit a pull request with a clear description
